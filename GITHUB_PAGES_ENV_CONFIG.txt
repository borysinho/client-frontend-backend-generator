# Configuración de Variables de Entorno para GitHub Pages

## Variables Configuradas

### 1. VITE_API_URL
- **Valor**: `https://server-frontend-backend-generator.vercel.app`
- **Estado**: ✅ Configurado en el workflow de GitHub Actions
- **Propósito**: URL del backend en producción

### 2. VITE_IA_API_KEY
- **Estado**: ⚠️ Requiere configuración manual en GitHub Secrets
- **Propósito**: API Key de Google AI (Gemini)

## Cómo Configurar Secrets en GitHub

### Opción A: Desde GitHub Web
1. Ve a tu repositorio: `https://github.com/borysinho/client-frontend-backend-generator`
2. Ve a **Settings** → **Secrets and variables** → **Actions**
3. Haz clic en **New repository secret**
4. Nombre: `VITE_IA_API_KEY`
5. Valor: Tu API key de Google AI
6. Haz clic en **Add secret**

### Opción B: Desde CLI (requiere GitHub CLI)
```bash
gh secret set VITE_IA_API_KEY --body "tu-api-key-aqui"
```

## Verificación

Después del próximo push a `main`, el workflow se ejecutará automáticamente y usará estas variables de entorno durante el build.

## Notas Importantes

- Las variables `VITE_*` son expuestas al cliente (frontend)
- Los secrets sensibles deben usar `secrets.*` en el workflow
- El archivo `.env` local no se commitea (está en `.gitignore`)
- Las variables del workflow sobrescriben las del `.env` durante el build
